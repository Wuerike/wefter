# Generated Artifacts

This document defines the target repository documents generated by the suite.

Default root:

```text
docs/wefter/
```

## Metadata Standard

Every generated markdown file should begin with:

```yaml
---
artifact: artifact-name
stage: discovery|product|architecture|module|task|ops
status: draft|human-review|approved|implemented|stale
owner_agent: agent-name
depends_on:
  - docs/wefter/path/upstream.md
feeds:
  - docs/wefter/path/downstream.md
human_review: required|optional|complete
last_updated: YYYY-MM-DD
language: <artifact_language>
---
```

`00-index.md` is the workflow state source of truth and uses additional setup metadata:

```yaml
---
artifact: wefter-index
stage: ops
status: active
workflow_mode: standard
artifact_language: pt-BR
artifact_root: docs/wefter
current_stage: discovery
active_module: null
complexity_default: medium
competitor_research_policy: offer-during-discovery
human_review: optional
last_updated: YYYY-MM-DD
---
```

## Status Values

- `draft`: created but not reviewed.
- `human-review`: requires human review before a downstream gate.
- `approved`: approved for downstream use.
- `implemented`: implemented in code.
- `stale`: upstream changes may have invalidated the artifact.

## Module Lifecycle

1. `module-map.md` selects module.
2. `module-brief.md` defines boundary.
3. `product-spec.md` defines behavior.
4. `technical-spec.md` defines implementation approach.
5. `testing-strategy.md` is created or updated when the module needs test approach changes.
6. `decisions.md` captures choices.
7. `tasks.md` is generated with a TDD plan and reviewed by a human.
8. Tasks are implemented one by one with red/green/refactor evidence.
9. Each task gets adversarial review, including test coverage and TDD compliance.
10. Module closes after all tasks pass and artifacts are current.

## Complexity Handling

The artifact tree stays stable across low, medium, and high complexity. Complexity changes depth, not filenames.

- Low can produce shorter sections and fewer alternatives, but still needs focused TDD or an explicit exception.
- Medium uses the standard templates.
- High adds deeper research, scoring, ADRs, risk traceability, and stronger review detail when relevant.
