# Scope

## Included

- Define technical-shaping inputs, outputs and stop conditions.
- Specify required technical artifacts, including decisions, constraints, data contracts and verification strategy.
- Define CLI run generation and validation requirements.
- Define OpenCode agents and permissions needed for a future available workflow.
- Keep future command names in `plannedCommands` until implementation is complete.

## Excluded

- Creating implementation task specs.
- Renaming delivery implementation workflows.
- Making product decisions or changing product scope.

## Constraints

- Technical shaping consumes product specs; it does not replace them.
- Unresolved product ambiguity must go back to product shaping or human decision.
- Architecture/security/persistence tradeoffs require explicit human gates.
